## What This Video Demonstrates

Claude Design is Anthropic's AI-native design tool that uses HTML/CSS/JS as its universal output format, letting a single tool generate animated videos, slide decks, landing pages, mobile app prototypes, and design systems — competing simultaneously with Figma, Gamma, Replit, Lovable, and Google Workspace.

## Workflow Steps

1. **Animated videos:** Paste content → Claude asks clarifying questions (length, tone, visual direction, format) → ~4-5 minute generation → HTML/CSS/JS animated video with tweak panel for color palette adjustments; export by screen recording (no native video export)
2. **Slide decks:** First generate an animated video from the content, then prompt "convert this video into an animated slide deck I can manually toggle through" — produces better results than asking for a static deck directly
3. **Landing pages:** Paste a URL or describe the page → Claude generates HTML/CSS landing page; can request tone variants (e.g., tongue-in-cheek) for copy exploration
4. **Mobile apps:** Describe the app → clarifying questions → ~4 minute generation → functional prototype with warm UI, exercise tracking, paywall stubs; Claude auto-takes screenshots and fixes bugs without being prompted
5. **Design systems:** Upload a Figma UI kit or design reference → Claude generates a typed design system (type scale, color tokens, spacing, motion, iconography, radii, shadows) → use the design system as context for subsequent app generation

## Claude Code Integration Points

- Claude Design's HTML/CSS/JS output files are directly editable in Claude Code — use Claude Code for the last 10% of manual fixes the video acknowledges AI cannot complete autonomously
- Design systems generated by Claude Design can be saved as project context files and passed to Claude Code for consistent component generation
- Claude Code's `sc:build` skill could take a Claude Design landing page and wire it to a real backend (forms, APIs, auth)
- The auto-screenshot-and-fix behavior Claude Design demonstrates is equivalent to Claude Code's verification loop — the same self-correction pattern applies when building in Code
- A workflow: Claude Design for visual prototype → export HTML → Claude Code to add interactivity, data fetching, and deployment

## Key Insight

**"Code is now the universal medium for knowledge work"** — by making HTML/CSS/JS the output format for videos, slides, apps, and design systems, Anthropic positions Claude Design to compete with every knowledge worker tool simultaneously, because all of these artifact types are ultimately just code.

## Reusable Primitives

- Video-first then slide deck: generate animated video from content first, then convert to togglable slides — produces structurally stronger decks than requesting slides directly
- Clarifying questions as a generation gate: Claude Design always asks before generating, which surfaces assumptions early and reduces iteration cycles
- Auto-screenshot-and-fix loop: Claude takes a screenshot of its own output and corrects visual bugs without user prompting — a built-in verification step
- Design system as generation context: extract tokens from a reference (Figma kit, existing app) then use as a style constraint for all subsequent outputs
- Tone variant exploration: same content, different tone instruction (professional vs. tongue-in-cheek) — useful for copy testing without rebuilding structure
- 90/10 rule: AI gets you to 90% reliably; budget time for the final 10% of manual refinement

## Limitations / Gotchas

- No native video export — animated video output requires screen recording, which adds a manual step and limits resolution control
- Design system quality is only as good as the reference input — an imperfect Figma kit (e.g., Apple Liquid Glass) produced a glassmorphic but buggy app output
- Mobile app prototypes include paywall stubs but no real payment integration — significant engineering work remains for production
- The video does not address hosting, deployment, or how to get Claude Design output onto a real domain
- Claude Design is a separate product from Claude Code — workflows that span both require manual file transfer between tools
- Competitor tools (Lovable, Replit) have more mature deployment pipelines; Claude Design's advantage is breadth of output types, not depth on any single one
