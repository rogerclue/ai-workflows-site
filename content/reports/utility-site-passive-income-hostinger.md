## What This Video Demonstrates

Shows how to build a Google Ads-monetized utility website (word unscrambler, calculator, etc.) in under an hour using Hostinger Horizons — a no-code AI website builder — then rank it on Google using a specific keyword research method that finds gaps where new sites are already breaking through. The model: build once, earn from display ads indefinitely.

## The Business Model

- **Revenue**: Google Display Ads (paid per 1,000 page views, not per click)
- **RPM benchmark**: $3–$12 per 1,000 page views; $9 RPM × 55M page views/mo = ~$500k/mo for top sites
- **Target**: Utility sites solving one specific evergreen problem (word unscramblers, calculators, IP tools)
- **Scale strategy**: Build a portfolio of 5–10 tools; $200/mo each = $1,000–$2,000/mo compounding

## Workflow Steps

### Phase 1 — Site Research: Find a Rankable Keyword
1. Pick a seed category (e.g., "generator", "calculator", "converter")
2. Open **Clearer** SEO tool → enter seed keyword → filter: max difficulty 10/100
3. Sort results by difficulty (easiest first) → identify queries that could be answered by a web app
4. Use Clearer's SERP analysis: flag queries where new/low-trust sites are already ranking on page 1
5. Blue-highlighted low-trust scores = Google is still looking for better results = opportunity
6. Target: high monthly search volume + difficulty ≤ 10 + new sites already ranking

### Phase 2 — Site Build: No-Code with Hostinger Horizons
1. Go to Hostinger Horizons → describe tool in plain English:
   *"Build a simple web app that is a word unscrambler. The user types in letters and it shows all real English words that can be made from those letters."*
2. Horizons generates working site in minutes
3. Iterate with plain-English fixes: "The letter box should be visible immediately" → done in 2 min
4. Add features by describing them: "Add options for starting letter, word length, must-contain letter"
5. UI improvements: move button closer, add color variation
6. **Publish**: click Publish button → live immediately on temporary domain

### Phase 3 — Domain Setup
1. Connect a custom domain matching the tool's function (e.g., wordunscramble[x].com)
2. Domain search built into Horizons; use ChatGPT for alternative name suggestions if taken
3. `.com` preferred for SEO; connect immediately — site goes live within minutes

### Phase 4 — Google AdSense Monetization
1. Go to `adsense.google.com` → sign up → enter site domain
2. Configure ad placements (preview on desktop + mobile; remove unwanted placements)
3. Copy AdSense code → paste into Hostinger Horizons chat: "Insert this code into the site"
4. Horizons inserts code and publishes changes automatically
5. Return to AdSense → verify site → request compliance review
6. **Before AdSense approval**: add legal pages (privacy policy, terms), 3–5 related tools, explanatory content about the site's purpose

### Phase 5 — Scale with Portfolio
1. Build multiple utility tools on the same domain OR separate sites
2. Publish each tool following the same process (under 1 hour each)
3. Compound: 5 tools × $200/mo each = $1,000/mo baseline before any breakout

### Bonus — Passive Referral Income via Remixable Templates
1. Inside any Horizons project → click "Let others copy your website" → Allow copying
2. Site becomes a remixable template for other Horizons users
3. If someone remixes your template and signs up for Horizons → you earn up to $150 referral commission per signup
4. More sites built = more templates = more passive referral potential

## Key Insight

**The keyword research method is the real moat** — finding queries where new, low-authority sites are already ranking on Google's page 1 is a direct signal that the search intent isn't being fully served. Building a utility site targeting those exact queries gives you the highest probability of fast ranking without an established domain or backlinks.

## Reusable Primitives

- **SERP gap analysis**: Use Clearer (or any SEO tool) to find queries where low-trust/new sites rank on page 1 → these are your build targets. Difficulty ≤ 10 + new sites ranking = actionable opportunity
- **Plain-English iterative UI build**: Describe the tool → get working version → iterate with natural language fixes ("move the button", "add this feature") → publish. No code review needed.
- **Portfolio compounding model**: Don't aim for one big hit. Build 5–10 small tools, each pulling independent traffic and ad revenue. One breakout tool changes the math entirely.
- **Referral template monetization**: Any no-code platform with template sharing can become a passive income stream — build → publish as template → earn referral commissions when others sign up through your template
- **AdSense via AI code injection**: "Insert [code] into this website" is a universal pattern for adding third-party scripts to AI-built sites — works for analytics, ads, chat widgets, etc.

## Limitations / Gotchas

- **AdSense approval is not instant**: New sites need legal pages, explanatory content, and related tools before Google approves them — plan 2–4 weeks of extra work before monetization kicks in
- **Hostinger Horizons is sponsored**: This video is sponsored content; Hostinger Horizons is the specific recommended tool, but Claude Code, Cursor, or Bolt.new could produce equivalent results for technical users
- **Traffic is the bottleneck, not the build**: Building takes under an hour; ranking on Google can take weeks to months even with good keyword selection. Most utility sites will earn nothing initially.
- **Clearer is a paid tool**: The specific SERP gap analysis method requires Clearer (or similar); free keyword tools don't surface the "new sites ranking" signal
- **$6M/year headline is aspirational**: The top-performing utility sites are long-established with millions of monthly visitors; a new site will earn far less, likely $0–$50/mo for the first 6–12 months
- **Mobile optimization matters**: Google prioritizes mobile-first indexing — verify the site looks correct on mobile immediately after publishing
